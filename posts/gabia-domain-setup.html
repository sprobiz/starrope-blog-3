가비아 개인 도메인 깃허브 Pages 연결 및 SSL/HTTPS 보안 설정 완벽 가이드

작성자: Starrope • 2026. 06. 03 • 읽기 시간: 약 5분
포스트 상단 Premium 배너

구글 애드센스 수익을 극대화하고 나만의 브랜드 가치를 높이기 위해 가장 먼저 실행해야 할 작업은 바로 '개인 커스텀 도메인 연결'입니다. 무료 기본 주소(예: github.io)는 애드센스 하위 도메인 승인이나 검색 최적화(SEO) 면에서 제약이 많기 때문입니다. 국내 대표 도메인 등록처인 가비아(Gabia)에서 산 개인 도메인을 깃허브 Pages 블로그에 연결하고, 흔히 겪는 'HTTPS Enforce 비활성화' 문제까지 깔끔하게 해결하는 전 과정을 상세히 정리해 드립니다.

1. 가비아(Gabia) DNS CNAME 레코드 세팅 요령

가장 먼저 해야 할 일은 내 도메인(`starrope2023.com`)으로 들어오는 트래픽이 깃허브 서버 호스팅 지점(`sprobiz.github.io`)을 찾아갈 수 있게 이정표를 세우는 것입니다. 가비아의 [DNS 관리툴] 페이지에서 연결할 블로그 서브도메인의 CNAME 레코드를 아래와 같이 입력합니다.

"가비아 CNAME 레코드 설정 시 값/위치 항목에 깃허브 주소를 적을 때는 끝에 마침표(.)를 반드시 붙여주어야 오류 없이 저장됩니다."

실전 CNAME 레코드 입력 규격 예시:

  • 타입: `CNAME`
  • 호스트: `blog3` (연결하고자 하는 3번 블로그의 서브도메인 이름)
  • 값/위치: `sprobiz.github.io.` (마침표 포함 필수)
  • TTL: `1800` (반영 주기를 30분 단위로 단축시켜 도메인 전파 속도를 극대화하는 꿀팁입니다.)

2. 깃허브 Pages CNAME 주입과 DNS 정상 인증 확인

레코드를 세팅했다면 깃허브 저장소에서도 이를 맞이할 준비를 해야 합니다. 저장소 루트 폴더에 소문자로 **`CNAME`**이라는 이름의 파일을 새로 생성하고, 그 안에 내가 연결할 도메인 주소(예: `blog3.starrope2023.com`)를 단 한 줄만 적어 저장하면 됩니다.

본문 내 네이티브 하이브리드 광고

이후 깃허브 저장소의 **[Settings] ➡️ [Pages]** 탭으로 들어가면 Custom domain 항목에 내가 설정한 도메인이 등록되어 있으며, 자동 테스트를 거쳐 **`DNS check successful` (DNS 검증 완료)** 초록색 가이드 메시지가 안전하게 나타나는 것을 볼 수 있습니다.

3. 초보자를 당황하게 하는 HTTPS 비활성화 해결법

도메인 연결 직후 많은 블로거들이 당황하는 지점이 바로 **`Enforce HTTPS` (보안 보안 연결 강제)** 체크박스가 grayed out 되어 비활성화되어 있는 현상입니다. 경고문에는 '도메인이 HTTPS를 지원하도록 제대로 구성되지 않았다'는 위협적인 문구가 뜨기도 합니다.

결론부터 말씀드리면 이는 **설정 오류가 아니라 Let's Encrypt를 통한 무료 SSL 보안 인증서가 깃허브 서버 측에서 발급되는 물리적 대기 시간** 때문입니다. DNS 검증이 성공했다면 백그라운드에서 보안 인증서 생성 스크립트가 실행되고 있는 중이므로, **약 5분에서 30분 정도** 여유를 가지고 기다리시면 됩니다. 대기 후 페이지를 새로고침하면 체크박스가 정상 활성화되며, 체크 시 `https://` 보안 통신 프로토콜이 최종 적용됩니다.

결론

개인 도메인을 연결하는 것은 애드센스 승인 심사를 한 번에 통과하고 고부가가치 타깃팅 광고 단가를 이끌어내는 데 있어 타협할 수 없는 필수 과제입니다. 이번 세팅을 마친 블로그는 향후 검색 노출 유입량이 폭발할 때 완벽한 자산 증식 플랫폼으로 기능하게 될 것입니다. 꼼꼼히 설정하셔서 보안 자물쇠 표시(HTTPS)가 달린 멋진 나만의 개인 브랜딩 사이트를 완성해 보시기 바랍니다.
